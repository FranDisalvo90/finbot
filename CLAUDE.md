# FinBot

Personal finance tracker. Monorepo with pnpm workspaces.

## Security Reviews

Static security audits are stored in `docs/security/`. Always read the latest review before working on:

- Authentication / authorization
- Tenant isolation (household / `householdId` boundaries)
- Sensitive data handling (expense descriptions, amounts, Splitwise tokens)
- Dependency upgrades or new third-party integrations

Latest review: [`docs/security/2026-06-08-security-review.md`](docs/security/2026-06-08-security-review.md)

Reviews are generated by the `/security-audit` skill from the Pluggy claude-plugins marketplace.

## Stack

- **Backend:** `packages/api` — Node.js + TypeScript + Hono + Drizzle (PostgreSQL)
- **Frontend:** `packages/dashboard` — React + Vite + TypeScript + Tailwind CSS + Recharts
- **AI:** Anthropic SDK (claude-sonnet-5) for expense categorization
- **DB:** PostgreSQL 16 via Docker Compose

## Commands

```bash
docker compose up -d        # Start Postgres
pnpm dev                    # Start API (port 3001) + Dashboard (port 5173)
pnpm lint                   # ESLint (flat config, root)
pnpm db:generate            # Generate Drizzle migrations
pnpm db:migrate             # Run migrations
pnpm db:seed                # Seed categories (6 parents + children with emojis)
pnpm db:migrate:test        # Run migrations on test DB
pnpm test:api               # Run API tests (uses finbot_test DB)
```

## Auth

- Google Identity Services (GSI) + JWT (HS256, 7-day expiry)
- Backend: `google-auth-library` verifies Google ID tokens, `hono/jwt` middleware
- Frontend: `@react-oauth/google`, `AuthProvider` context in `src/lib/auth.tsx`
- Users auto-created on first Google login
- Auth middleware applied to all `/api/*` except `/api/auth` and `/api/health`
- Env vars: `GOOGLE_CLIENT_ID`, `JWT_SECRET`, `VITE_GOOGLE_CLIENT_ID`

## Key Paths

- Schema: `packages/api/src/db/schema.ts` (categories, expenses, categorization_rules, imports, users)
- Routes: `packages/api/src/routes/` (auth, categories, expenses, reports, import, rules)
- Parsers: `packages/api/src/services/parsers/` (visa-galicia.ts, splitwise.ts)
- Categorizer: `packages/api/src/services/categorizer.ts`
- Pages: `packages/dashboard/src/pages/` (Login, Dashboard, Import, Categorize, Categories)
- Test helpers: `packages/api/src/test/helpers.ts`

## Conventions

- UI is in Spanish (Argentine context)
- Money formatted as ARS with `Intl.NumberFormat('es-AR')`
- Self-referencing FK in categories uses `AnyPgColumn` type for Drizzle TS compat
- Tests use vitest with a `finbot_test` DB (port 5433)
- Always run `pnpm lint` and `pnpm test:api` before pushing
